새 기능을 구현할 계획을 세워줘.

기능 설명: $ARGUMENTS

다음 단계로 진행해줘:

1. **영향 범위 분석**: 관련 파일 목록, 수정이 필요한 기존 코드 파악
2. **구현 계획**: 어떤 파일을 생성/수정할지 단계별로 정리
3. **데이터 모델 변경 여부**: Article/Label 모델 변경이 필요한지 확인
4. **UI 문자열**: 필요한 ARB 키 목록 (ko/en)
5. **디자인 시스템 준수**: 기존 AppColors, Spacing, Radii 등 토큰 활용 계획

계획을 보여주고 승인 후 구현을 시작해.
