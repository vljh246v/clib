변경된 코드를 분석하고 커밋 후 푸시해줘.

1. `git status`로 변경된 파일 확인
2. `git diff`로 변경 내용 분석
3. `git log --oneline -5`로 최근 커밋 스타일 확인
4. 변경 내용을 분석하여 적절한 한국어 커밋 메시지 작성
5. 커밋 메시지와 변경 파일 목록을 보여주고, 사용자에게 "커밋 후 푸시할까요?" 라고 확인 요청
6. 승인 시 git add → git commit → git push 실행
7. 거부 시 작업 중단

주의사항:
- 커밋 메시지는 한국어, 한 줄 요약 형식
- .env, credentials 등 민감 파일은 커밋하지 않음
- 변경 파일이 많거나 성격이 다르면 분리 커밋 권장
- 반드시 사용자 승인 후에만 커밋/푸시 진행
